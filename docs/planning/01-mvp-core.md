# 01. MVP核心需求与流程

更新时间: 2026-04-18

## 0. 项目形态锁定

1. 本项目为网页项目(Web)，仅通过浏览器访问。
2. 本阶段不开发桌面客户端、移动原生客户端和小程序端。

## 1. 核心角色

| 角色 | 核心目标 | 关键权限 |
|---|---|---|
| 普通用户(买卖一体) | 发布闲置、浏览商品、沟通并完成交易 | 注册登录、发布/编辑商品、发起沟通、创建交易、确认成交、取消交易 |
| 管理员 | 保障平台内容与交易秩序 | 审核商品、下架违规内容、处理举报、冻结用户(极简风控) |

## 2. MVP边界

### 2.1 范围内(必须做)

1. 用户注册登录(单一方式即可)
2. 商品发布与管理(发布、编辑、下架)
3. 商品浏览(列表、详情、基础筛选)
4. 买卖双方沟通(站内会话)
5. 交易闭环(发起交易意向、双方确认、成交/取消)
6. 管理员审核(通过/拒绝)

### 2.2 范围外(本期砍掉)

1. 在线支付与担保
2. 物流发货系统
3. 推荐算法与个性化首页
4. 收藏、点赞、评论体系
5. 复杂信用分、积分、会员体系
6. 非网页端形态(桌面客户端、移动原生客户端、小程序)
7. 高级风控模型

## 3. MVP功能清单

1. 账号模块: 注册、登录、退出、基础资料
2. 商品模块: 发布、图片上传、上架状态管理、详情展示
3. 浏览模块: 列表、关键词搜索、分类筛选、价格排序
4. 沟通模块: 基于商品的买卖双方会话
5. 交易模块: 发起交易、约定价格与地点、双方确认完成、取消
6. 审核模块: 管理员审核商品并记录原因

## 4. 核心业务流转

卖家发布商品
-> 系统置为待审核
-> 管理员审核通过
-> 商品上架可见
-> 买家浏览并进入详情
-> 买家发起沟通
-> 双方协商并发起交易
-> 卖家确认交易意向
-> 线下完成交付
-> 买家/卖家双方确认完成
-> 商品状态变为已售出(闭环完成)

### 关键异常分支

1. 审核拒绝: 待审核 -> 审核拒绝(卖家可修改后再次提交)
2. 交易取消: 交易中 -> 已取消(商品回到在售)
3. 主动下架: 在售 -> 已下架(不可被新会话发起)
