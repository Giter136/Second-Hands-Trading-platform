# 04. .gitignore核心规则分类说明

更新时间: 2026-04-18

## 1. 目标

1. 防止敏感信息入库。
2. 防止依赖与构建产物污染版本历史。
3. 防止 AI 临时草稿与中间文件进入仓库。

## 2. 必须排除的核心分类

## 2.1 环境与密钥类

必须排除:

1. 环境变量文件: .env, .env.local, .env.*
2. 私钥与证书: *.key, *.pem, *.crt
3. 本地密钥目录与临时密钥文件

原因: 避免凭据泄漏与安全事故。

## 2.2 依赖与虚拟环境类

必须排除:

1. 前端依赖目录: node_modules
2. Python虚拟环境目录: .venv, venv
3. Python缓存文件: __pycache__, *.pyc

原因: 体积大、可重建、无版本价值。

## 2.3 构建产物与覆盖率类

必须排除:

1. 前端构建目录: .next, dist, build
2. 测试覆盖率目录与报告文件
3. 临时缓存目录与中间产物

原因: 自动生成内容不应进入源码版本管理。

## 2.4 运行日志与进程文件类

必须排除:

1. 日志文件: *.log
2. 进程与锁文件: *.pid, *.lock
3. 运行时临时目录: tmp, temp

原因: 高频变化、无协作价值。

## 2.5 编辑器与系统噪音文件类

必须排除:

1. 操作系统缓存文件
2. 编辑器临时文件与本地会话文件
3. 本地调试快照

原因: 与业务无关，易造成无效冲突。

## 2.6 数据与备份类

必须排除:

1. 本地数据库文件: *.db, *.sqlite3
2. 数据库导出备份文件
3. 大体积本地测试数据集

原因: 数据可由迁移与种子流程重建，不应直接纳入主仓库。

## 2.7 AI临时草稿与实验文件类

必须排除:

1. AI 生成的草稿目录与暂存目录
2. 带有 draft、scratch、tmp 标识的临时文件
3. 中间分析报告与一次性验证文档

原因: 防止 AI 试探性产物污染主线历史，保证仓库整洁。

## 3. 落地执行要求

1. .gitignore 规则由人类维护，AI仅可提议变更。
2. 每次新增工具链时，先更新忽略规则再开始开发。
3. 发现误提交敏感文件时，立即执行应急处理并轮换相关密钥。
